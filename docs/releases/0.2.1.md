# Release Notes: 0.2.1

## Highlights

This release tightens the 0.2.0 baseline: it closes several locale/runtime edge cases, improves visitor-facing accessibility and UX, and aligns public docs with the current routing and config model.

## Major Themes

### i18n and config correctness

- fixed locale path stripping so non-locale slugs that merely start with locale letters are no longer mangled
- preserved region-code locale casing in the `new-post` fallback parser
- clarified locale resolution behavior and removed dead locale-validation config surface
- fixed localized home-hero fallback behavior so missing hero copy can still fall back cleanly

### UX and accessibility polish

- kept the language switcher visible on mobile
- added focus entry, focus trapping, and focus restoration for About page modals
- extended reading toast visibility to improve readability
- clarified “Regenerate” interactions as replay/scan effects rather than content-changing actions
- marked derived AI metrics as estimates instead of presenting them like measured values

### Runtime and tooling hardening

- escaped JSON-LD output more safely in the document head
- reduced RSS output to explicit feed fields rather than spreading full frontmatter payloads
- upgraded adapter contract checks toward import/behavior validation instead of relying mainly on raw string matching
- kept starter adapter templates aligned with the updated runtime/config surface

### Docs and submission alignment

- updated README and listing copy to reflect the current default-locale routing baseline
- replaced outdated config terminology with the current `theme.enableAboutPage` surface
- clarified that `npm update @anglefeint/astro-theme` updates the published package first and does not automatically rewrite starter-managed app files

## Migration Notes

- no new config migration is required on top of the 0.2.0 baseline
- starter sync is still required for this release because starter-consumed runtime/config files and docs changed together

## Starter Sync

- starter sync is required for this release

- `node scripts/check-scaffold.mjs`
