# Release Notes: 0.2.2

## Highlights

This release moves the cyber pagination UI out of the starter app layer and into `@anglefeint/astro-theme`, aligning pagination ownership with the rest of the package-owned theme system.

## Major Themes

### Package-owned pagination

- moved `CyberPagination.astro` into the published theme package
- moved pagination model and style helpers into package-owned utilities
- updated the blog list route to consume package subpath imports instead of starter-local pagination files

### Starter boundary cleanup

- removed the starter-owned pagination component and helper files from the managed starter surface
- updated the starter manifest so future starter syncs remove the old pagination files instead of keeping duplicate implementations

### Local development resolution

- added workspace TypeScript path mappings for `@anglefeint/astro-theme/components/*` and `@anglefeint/astro-theme/utils/*`
- kept local type-checking aligned with both workspace-source and installed-package package subpath imports

## Migration Notes

- no config migration is required
- starter sync is required for this release because the starter branch must delete the old pagination files and pick up the updated blog-list import path
- existing starter consumers created before this release will not pick up package-owned pagination by running `npm update` alone; they need the updated starter-side route/import surface

## Starter Sync

- starter sync is required for this release

## Validation Summary

- `npm run check:adapters`
- `npm run check`
- `node tools/maintainer/sync-starter.mjs --check`
