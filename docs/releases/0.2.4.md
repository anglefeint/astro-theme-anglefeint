# Release Notes: 0.2.4

## Highlights

This release tightens the starter user experience. It removes misleading config surface, adds a focused in-site guide series, and narrows starter content distribution to an explicit onboarding whitelist.

## Major Themes

### Config surface cleanup

- made `site.description` the site-level default description fallback for the home page
- removed the unused locale `meta.dir` field from the starter config contract instead of leaving partial RTL surface exposed
- updated user-facing README guidance so the supported site and locale metadata fields match actual runtime behavior

### Starter onboarding content

- added a three-part localized starter guide series:
  - site configuration
  - languages and routing
  - comments, About, and theme toggles
- kept the guide content grounded in fields that are actually consumed by the current starter/runtime implementation
- kept `welcome-to-anglefeint` as the lightweight entry post alongside the new guide series

### Starter sync contract hardening

- changed starter blog distribution from an implicit full-content carryover to a whitelist-driven sync model
- defined the starter onboarding post set in `scripts/starter-manifest.mjs`
- updated `release:starter` so it syncs only the approved onboarding posts and removes extra starter-side articles outside the whitelist

## Migration Notes

- no new migration is required for existing starter users beyond the usual package/starter update flow
- starter sync is required for this release because starter-distributed content and starter sync contract behavior changed together
- maintainers should treat the starter blog set as whitelist-driven going forward rather than assuming all `main` posts belong in `starter`

## Starter Sync

- starter sync is required for this release
