# Release Notes: 0.2.7

## Highlights

This release keeps the theme behavior stable while reducing maintenance friction around SEO/head derivation, deferred post effects, and generated adapter boundaries.

## Major Themes

### Lightweight runtime cleanup

- extracted blog post deferred-start timing values into named constants instead of inline magic numbers
- moved BaseHead locale/alternate/schema derivation into a small shared helper while preserving the rendered head contract
- centralized theme-default i18n entry resolution in `astro.config.mjs` behind a single helper

### Stronger adapter boundaries

- added explicit generated-file warnings to starter adapter targets under `src/config/*` and `src/i18n/*`
- updated `sync-adapters` output so local edits to generated adapter files are called out as overwrite-prone
- added the new config helper to the starter manifest so starter sync continues to produce a valid template

## Migration Notes

- existing users can upgrade by updating `@anglefeint/astro-theme`; no content migration is required
- no `src/site.config.ts` changes are needed for this release
- starter sync is required so new starter users receive the updated helper and generated adapter warnings

## Starter Sync

- starter sync is required for this release

## Validation Summary

- `npm run lint`
- `npm run check:no-build`
- `npm run maintainer:sync-starter:check`
